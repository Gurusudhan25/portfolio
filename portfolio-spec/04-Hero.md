# Hero Section Specification

## Objective

The Hero section is the most important section of the portfolio.

It should immediately create a strong first impression and communicate engineering excellence.

Within the first 5 seconds, visitors should understand:

- Who I am
- What I do
- What kind of engineer I am
- Why they should continue exploring

The Hero section should feel like the opening scene of a premium product launch rather than a traditional developer portfolio.

# Background

Do NOT use a static background.

The background should feel alive.

Use multiple layers.

Suggested layers:

• Large blurred gradient orbs

• Soft moving gradients

• Noise / grain texture

• Very subtle grid

• Small glow accents

• Floating blurred shapes

The background must remain minimal.

It should never distract from the content.

---

# Hero Portrait

Use the uploaded profile image.

Never display the original image directly.

Generate multiple artistic versions.

Recommended style:

- Remove background
- Improve lighting
- Premium color grading
- Soft shadows
- Gradient overlay
- Slight glass reflection
- Rounded presentation
- Elegant border

Possible effects

• Monochrome portrait

• Gradient tint

• Glassmorphism frame

• Floating portrait card

• Circular masked portrait

• Premium cutout

The portrait should appear like part of the design rather than simply being placed on the page.

---

# Headline

The headline should be bold.

Minimal.

Readable.

Professional.

Examples

Software Engineer

Building scalable web applications with thoughtful engineering.

Alternative

Software Engineer focused on performance, clean architecture, and enterprise-grade frontend development.

Alternative

Designing and building modern software experiences.

Do NOT use flashy marketing language.

# Performance

The Hero should load quickly.

Avoid heavy JavaScript.

Prefer CSS transforms.

Lazy-load non-critical assets.

Maintain smooth 60fps animations.

---

# Accessibility

Proper heading hierarchy.

Keyboard-accessible buttons.

Screen-reader friendly.

Respect reduced-motion settings.

Maintain sufficient contrast.

---

# Mobile Experience

The Hero should be redesigned for mobile.

Portrait centered.

Typography scaled.

Buttons full-width where appropriate.

Statistics stack vertically.

Spacing remains generous.

Nothing should feel cramped.

---

# Premium Details

Include subtle details such as:

• Gradient text highlights

• Animated underline

• Glass floating badge

• Cursor interaction (desktop)

• Soft glowing borders

• Smooth button hover

• Slight portrait elevation

• Layered depth

These details should enhance the experience without becoming distracting.

---

# AI Instructions

The Hero must feel handcrafted.

Avoid generic portfolio layouts.

Avoid template-like structures.

The first impression should communicate quality, confidence, and engineering excellence.

A recruiter should immediately think:

"This is someone who builds premium products."

The Hero should set the visual standard for the rest of the portfolio.
