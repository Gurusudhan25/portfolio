# Design System

## Overview

The design system should feel like a premium SaaS product rather than a personal portfolio.

Every component should follow a consistent visual language.

The overall feeling should be:

- Premium
- Modern
- Minimal
- Elegant
- Engineering-focused
- Interactive
- High-end

The interface should immediately communicate quality and attention to detail.

---

# Design Inspirations

Take inspiration from the following products:

- Apple
- Linear
- Stripe
- Vercel
- Framer
- Raycast
- Notion
- Figma

Do NOT replicate their designs.

Instead, combine their design philosophies into a unique visual identity.

---

# Visual Style

The website should use:

- Clean layouts
- Large whitespace
- Rounded cards
- Soft shadows
- Layered sections
- Glassmorphism (used sparingly)
- Beautiful gradients
- Subtle grain textures
- Smooth transitions

Avoid:

- Heavy glow effects
- Neon cyberpunk themes
- Generic developer portfolio styles
- Matrix effects
- Excessive particle backgrounds
- Cluttered layouts
- Loud colors

# Spacing System

Use an 8-point spacing system.

Examples:

8

16

24

32

48

64

96

128

Large breathing space between sections.

Avoid cramped layouts.

---

# Grid System

Desktop

12-column layout

Tablet

8-column layout

Mobile

Single column

Content width should remain readable.

Never stretch text too wide.

---

# Border Radius

Cards

Large rounded corners.

Buttons

Rounded.

Inputs

Rounded.

Images

Rounded.

Avoid sharp corners.

---

# Shadows

Use soft layered shadows.

No harsh shadows.

Cards should appear elevated naturally.

---

# Borders

Very subtle.

Low opacity.

Used only when necessary.

---

# Buttons

Primary Button

Filled

Rounded

Medium height

Smooth hover animation

Slight elevation

---

Secondary Button

Outlined

Transparent

Border only

---

Text Button

Minimal

Underline animation

---

# Cards

Cards are a major part of the design.

Each card should have:

Rounded corners

Soft border

Layered background

Hover elevation

Smooth shadow transition

Micro animation

Cards should never feel flat.

---

# Icons

Use a consistent icon library.

Recommended:

Lucide Icons

Icons should remain simple and clean.

Avoid filled icons.

---

# Images

Images should never appear as plain rectangles.

Apply:

Rounded corners

Soft shadows

Subtle overlays

Elegant presentation

---

# Glass Effects

Glassmorphism should be used carefully.

Only for:

Navigation

Floating panels

Hero highlights

Do not overuse glass effects.

---

# Background

The background should feel alive without being distracting.

Use:

Soft gradients

Blurred shapes

Noise texture

Animated gradient movement

Subtle lighting

Avoid:

Busy illustrations

Random particles

Heavy graphics

---

# Animations

Animations should feel natural.

Never flashy.

Every animation should improve the experience.

Use:

Fade

Slide

Scale

Blur

Parallax

Opacity

Rotation (minimal)

Avoid:

Bounce

Elastic effects

Crazy rotations

Long animations

---

# Hover Effects

Cards

Lift slightly

Shadow increases

Border glows softly

---

Buttons

Gradient movement

Shadow

Scale 1-2%

---

Links

Underline animation

Smooth color transition

---

# Scroll Animations

Sections reveal naturally.

Use:

Fade In

Slide Up

Scale

Blur Reveal

Cards should animate one after another.

Not all at once.

---

# Loading Experience

Initial page load should feel premium.

Ideas:

Soft fade

Logo animation

Gradient reveal

Content stagger

Keep loading under one second whenever possible.

---

# Cursor

Desktop only.

Optional custom cursor.

Should be subtle.

Interactive elements should react smoothly.

Do not make the cursor distracting.

---

# Responsive Design

Every layout should be designed individually.

Do not simply shrink desktop layouts.

Mobile should feel intentional.

Spacing should remain generous.

Typography should scale beautifully.

---

# Component Consistency

Every component should share:

Spacing

Radius

Animation speed

Shadow style

Typography

Color usage

Hover behavior

Transition timing

---

# Accessibility

High color contrast

Keyboard navigation

Visible focus states

ARIA labels where appropriate

Reduced motion support

Semantic HTML

---

# Performance

Avoid unnecessary CSS.

Avoid large animation libraries where possible.

Prefer CSS transforms.

Lazy load images.

Optimize every asset.

Keep animations GPU accelerated.

---

# Final Impression

The website should feel like a premium software product built by an engineer who cares deeply about quality, performance, and user experience.

A recruiter should think:

"This person builds products, not just websites."
